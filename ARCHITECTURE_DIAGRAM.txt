TESTING MODES
─────────────

Mode 1: SIMULATION (--simulate flag)
├─ No BPF required
├─ No root required
├─ Synthetic dwell patterns
├─ Tests ADMM algorithm convergence
└─ Perfect for development/CI

Mode 2: REAL BPF (default)
├─ Requires root + CAP_BPF
├─ Requires Ubuntu 25.10+ kernel
├─ Actual file system monitoring
├─ Tests full system integration
└─ Production-grade validation

Mode 3: INTEGRATION TEST (test-suite.sh)
├─ Orchestrates both modes
├─ Validates end-to-end
├─ Checks metrics
├─ Reports results
└─ Fastest way to validate everything
